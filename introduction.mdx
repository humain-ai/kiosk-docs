Humain Kiosk is a multi-channel AI conversation platform. Your team configures AI-powered personas,
uploads knowledge documents, and sets behavioural guardrails in the admin panel. Physical devices —
terminals, VoIP adapters, website widgets — authenticate with per-device API tokens and talk to users.
All intelligence runs server-side. Devices are thin clients.

<Tip>
  Use **sandbox mode** (`https://sandbox.api.humain.ai`) during development. Responses are
  deterministic, no AI credits are consumed, and billing events are not generated. Switch to
  the production URL when you are ready to go live.
</Tip>

## Quick start

<Steps>
  <Step title="Configure a kiosk">
    Sign in to the [admin panel](https://app.humain.ai), create a workspace, and add a kiosk.
    Give the kiosk a persona, attach knowledge documents, and set your guardrail rules.
  </Step>
  <Step title="Issue a device credential">
    Go to **Kiosk → Credentials → Issue new credential**. Copy the token — it is shown
    only once. Store it in a secret manager immediately. Your device uses this token in every
    `Authorization: Bearer` header.
  </Step>
  <Step title="Open a session">
    Call `POST /v1/sessions` with `{"mode":"text"}` to start a conversation. Then call
    `POST /v1/sessions/{id}/message` for each user turn, and `POST /v1/sessions/{id}/end` when
    done. See [Try it out](/playground/text) for a live interactive demo.
  </Step>
</Steps>
