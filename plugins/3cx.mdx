<Note>
  This page is for 3CX administrators who want to route calls to the Humain AI using the 3CX
  management interface. For programmatic control via SIP trunk without the plugin, see the
  [SIP gateway](/voip/sip-gateway) guide instead.
</Note>

The **Humain Kiosk** 3CX app is a **CallFlow application** that appears as a destination in the
3CX call routing editor. Route any inbound DID, IVR option, or queue overflow to the Humain AI
— configured entirely from the 3CX Management Console.

---

<Steps>
  <Step title="Install the app from the 3CX App Marketplace">
    In the **3CX Management Console**, go to **Integrations → 3CX App Marketplace**. Search for
    **"Humain Kiosk"** and click **Get App**.

  </Step>

  <Step title="Configure your credential">
    After installation, go to **Integrations → 3CX Apps → Humain Kiosk → Settings**. Paste your

    <Note>
      Issue credentials from the Humain admin panel under **Kiosk → Credentials → Issue new
      credential**. The token is displayed only once — copy it before closing that page.
    </Note>
  </Step>

  <Step title="Add a Humain Kiosk CallFlow destination">
    Go to **Call Routing → CallFlow**. Click **Add App Instance** and select **Humain Kiosk**.
    Configure the instance:

    - **Name** — a label for this CallFlow destination (e.g., "AI Receptionist")
    - **Codec** — G.722 recommended; G.711 PCMU for legacy carrier compatibility
    - **Maximum call duration** — timeout in seconds (0 = unlimited)
    - **Fallback extension** — where to transfer if the AI session cannot be opened

    Click **OK**, then **Save** the CallFlow.
  </Step>

  <Step title="Route calls to the AI">
    With the CallFlow instance saved, you can route calls to it from anywhere in 3CX:

    <Tabs>
      <Tab title="Inbound rule">
        Go to **Call Routing → Inbound Rules**. Edit the rule for your DID number. Set the
        **Route calls to** destination to your Humain Kiosk CallFlow instance.

## Advanced settings

<AccordionGroup>
  <Accordion title="Caller metadata">
    When **Pass caller metadata** is enabled (default: on) in the app settings, the plugin
    passes the following fields to each Humain Kiosk session:

    | Metadata key | Value |
    |---|---|
    | `caller_id` | Caller CLI (ANI) |
    | `did` | Dialled DID (DNIS) |
    | `queue_name` | Source queue name, if routed from a queue |
    | `wait_time_seconds` | Time the caller spent waiting before AI answered |

    Your kiosk's action endpoints can use this data to personalise the response — for example,
    greeting callers by name after a CRM lookup.
  </Accordion>

  <Accordion title="Transferring back to a live agent">
    The Humain AI can transfer a call back to a 3CX extension or queue using the **Transfer**
    action defined in your kiosk's action endpoints. Configure the target extension number in
    the action endpoint's response payload; the 3CX plugin will execute a supervised transfer.

    See the admin panel (**Kiosk → Actions**) for configuring action endpoints.
  </Accordion>

  <Accordion title="Supported 3CX versions">
    | 3CX version | Support |
    |---|---|
    | 3CX v20 (latest) | ✅ Fully supported |
    | 3CX v18 | ✅ Supported |
    | 3CX v16 | ⚠ Limited — CallFlow apps require v18+ for some features |
    | 3CX v15.5 and earlier | ❌ Not supported |
  </Accordion>
</AccordionGroup>
