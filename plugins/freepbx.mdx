<Note>
  This page is for FreePBX administrators who want to route calls to the Humain AI without
  writing dialplan code. If you are an Asterisk developer who prefers scripted control, see the
  [Asterisk AGI integration](/voip/asterisk-agi) guide instead.
</Note>

The **Humain Kiosk** FreePBX module adds a new destination type — **Humain Kiosk AI** — to your
FreePBX routing table. Once installed, you can route any inbound route, IVR option, or ring
group timeout directly to the AI, just like routing to a queue or voicemail.

---

## Installation

<Steps>
  <Step title="Install the module">
    In your FreePBX admin panel, go to **Admin → Module Admin**. Click **Check Online** to
    refresh the module repository, then search for **"Humain Kiosk"**. Click **Download** and
    then **Process**.

    [FreePBX module page](https://www.freepbx.org/add-ons/add-on-modules/humain-kiosk/) and
    upload it via **Module Admin → Upload**.

    After installation, click **Apply Config** to reload Asterisk.
  </Step>

  <Step title="Enter your credential">
    Go to **Admin → Humain Kiosk Settings**. Paste your device credential token into the
    **Credential** field and click **Save**.

    <Note>
      Issue credentials from the Humain admin panel under **Kiosk → Credentials → Issue new
      credential**. The token is displayed only once — copy it before closing that page.
    </Note>
  </Step>

  <Step title="Configure the AI destination">
    Go to **Connectivity → Humain Kiosk Destinations**. Click **Add Destination** and configure:

    - **Name** — a label for this destination (e.g., "Main AI Agent")
    - **Codec preference** — G.722 recommended; G.711 PCMU/PCMA for legacy carriers
    - **Timeout** — maximum call duration in seconds (0 = unlimited)
    - **Fallback destination** — where to route the call if the AI is unavailable

    Click **Save**, then **Apply Config**.

  <Step title="Route calls to the AI">
    With the destination created, route calls to it from anywhere in FreePBX:

    <Tabs>
      <Tab title="Inbound route">
        Go to **Connectivity → Inbound Routes**. Edit the route for the DID number you want the
        AI to answer. Set **Set Destination** to **Humain Kiosk → [your destination name]**.

    response. Once satisfied, disable sandbox mode and apply config.

## Advanced settings

<AccordionGroup>
  <Accordion title="Passing caller ID to the AI session">
    When enabled, the module passes the caller's CLI (Caller Line Identification) and the
    dialled DID to the Humain session as metadata. Your kiosk's action endpoints can use this
    to look up customer records.

    Enable **Pass caller metadata** in **Admin → Humain Kiosk Settings → Advanced**. The
    following fields are added to each session:

    | Metadata key | Value |
    |---|---|
    | `caller_id` | Caller CLI (ANI) |
    | `did` | Dialled number (DNIS) |
    | `channel` | Asterisk channel name |
  </Accordion>

  <Accordion title="Codec configuration">
    The module negotiates codecs with the Humain SIP gateway in this preference order:
    G.722 → G.711 PCMU → G.711 PCMA.

    To force a specific codec, change the **Codec preference** setting on each destination.
    See the [codec guide](/voip/codec-guide) for quality recommendations.
  </Accordion>

  <Accordion title="Fallback on API error">
    If the Humain API is unreachable when the call arrives, the module routes to the configured
    **Fallback destination** instead of dropping the call. Set the fallback to a queue, voicemail,
    or another extension in **Connectivity → Humain Kiosk Destinations**.
  </Accordion>
</AccordionGroup>
