VoIP mode requires a SIP trunk, PBX, or ATA adapter — none of which run in a browser. Use one of
the tools below to test your VoIP integration locally before connecting to production.

## Minimal cURL test

You can still exercise the VoIP session lifecycle from the command line — only the audio bridge
requires telephony hardware:

```bash
# 1. Open a VoIP session — use an hk_test_ credential to avoid billing (see /concepts/sandbox-mode)
RESPONSE=$(curl -s -X POST https://api.humain.ai/v1/sessions \
  -H "Authorization: Bearer hk_test_your_credential_here" \
  -H "Content-Type: application/json" \
  -d '{"mode":"voip"}')

SESSION=$(echo "$RESPONSE" | jq -r .session_id)
AGENT_URL=$(echo "$RESPONSE" | jq -r .agent_url)

echo "Session ID: $SESSION"

# 2. End the session (simulating a hang-up) — on agent_url, not necessarily the same host
curl -s -X POST "$AGENT_URL/v1/sessions/$SESSION/end" \
  -H "Authorization: Bearer hk_test_your_credential_here"

echo "Session closed."
```

This verifies that your credential is valid and that session open/close works correctly before you
wire up the SIP bridge.

## C library voice sessions

WebRTC voice sessions opened via `kiosk_voice_session_open()` cannot be tested from a browser.
The `libdatachannel` ICE stack performs UDP hole-punching and DTLS handshakes that require
a real network socket — not available in a browser sandbox.

To test a voice session from your workstation before deploying to hardware:

1. Install the library locally (`cmake -B build -DKIOSK_WITH_VOICE=ON && sudo cmake --install build`).
2. Compile and run the minimal voice example from the [Raspberry Pi quickstart](/embedded/raspberry-pi#voice-session-quickstart) on your Linux development machine.
3. Use a USB headset as the ALSA capture and playback device.

```bash
export HUMAIN_CREDENTIAL=hk_live_your_credential_here
gcc pi-voice.c -lkiosk -lcurl -o pi-voice
./pi-voice
# [connecting]
# [listening]
# Voice session active — speak into the microphone. Press Ctrl+C to stop.
```

This verifies ICE connectivity, credential validity, and the full audio round-trip before you
deploy to a Raspberry Pi or embedded Linux target.
